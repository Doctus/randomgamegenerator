Introduction

RGG is a whiteboad client, primarily designed to facilitate play of traditional pen-and-paper role-playing games online, although it's usable for a variety of other types of board games and similar purposes.

If you have any questions or comments, you can probably get a hold of us most easily in one of these places:
Attercop Central
irc.darkmyst.org#attercop

Getting started

Of course, the first step is downloading RGG.

Windows

RGG is available in pre-compiled packages for Windows users, but if for some reason you choose to compile it yourself, follow these instructions.

Acquire the following:

  • pyQT
  • Python 2.6
  • MSYS (optional for glmod)
  • MinGW (optional for glmod)
  1. OPTIONAL OPTION 1: Move glmod.pyd or glmod_WIN64.pyd from module/windoze to scripts, renaming it to glmod.pyd in the latter case.
  2. OPTIONAL OPTION 2: Navigate to trunk/module in MSYS. Run build-windows.sh. Move resulting glmod.pyd to the scripts folder.
  3. Navigate to scripts and run rgg.py.

Mac OS X

Mac is not really supported; it's probably possible to get RGG working on it, but none of the dev team have one to test. Sorry!

Linux

  1. Obtain the libraries listed under Windows, above. The most recent Ubuntu repository versions should work.
  2. OPTIONAL: Navigate to module and run build.sh. Move resulting file to scripts.
  3. Navigate to scripts and run rgg.py.

Getting Started

Now that you’ve installed a shiny new Random Game Generator, what to do with it?
Of course you play games with it…but how?

First, you should familiarize yourself with the chat interface.

Chat functions

There are two chat windows in RGG: the in-character chat, and the out-of-character chat. You can use both at once and they're logged to the same file.

The in-character chat is used for communication that’s part of the game: the narration, characters talking to each other, combat, and so on. Out-of-character chat is used for things you’re saying as you, outside of the game: e.g., that you need to take a break to answer the phone, or debating the rules with other players. Using either chat window is as simple as typing your message and pressing Return, but there are some slightly more complex functions most users find invaluable.

Some of these basic chat functions include:

  • /roll xdy: rolls x dice with y sides.
  • /proll xdy, in the OOC window, will roll x dice with y sides and show the result to you alone. This is a handy tool for the GM.
  • /me x or /emote x: displays text as 3rd person narration, in the form of [name] x. (E.g., /me shrugs would print [name] shrugs.) If used in the IC chat window, the name used will be the currently selected character’s name; OOC chat will use your username.
  • /whisper X y: sends a message with the text y to X and no one else.

Creating characters

A game is not so interesting without characters. How to create characters to communicate in the IC chat window?

  1. Press the “Add new” button below the text entry in the IC chat window. A new window will appear.
  2. Fill in the appropriate fields. “List ID” is the name that will appear in your character selection drop-down menu (to the left of the “Add new” button). “Name” is the name that other players will see as the speaker of your character’s lines. “Portrait” is the picture that will appear next to your character’s dialogue. (If you do not want a portrait to appear, type one blank space in this box.) RGG checks the portraits folder for portraits — place whatever picture you want to use as a portrait in the folder before opening the new character dialogue.
  3. Make sure that the new character is selected in the drop-down menu, and begin typing.

You can keep an arbitrary number of characters in the list. If you have portraits representing several emotions of the same character, you could even create multiple "characters" to indicate their different moods.

Creating maps

Now that your characters have been made, of course they’ll need a world in which to go adventuring. You can create maps with the File > New Map command.

Enter the name of the map, your name, the number of tiles in each direction that you want the map to be, the tileset to use, and how many pixels large the individual tiles are in that set.

You can create an arbitrary number of maps, too, and move pogs freely between them. However, extremely large maps or having a huge number open at once can cause problems, so be careful.

Once your map has been created, you can edit it with the Map Editor. (Right-click on the top bar and tick the mark next to map editor.) Open the Map Editor and click on the map you want to edit. You can select the specific tile you want to place by clicking on it from the tile palette. Select whether you want to place one tile at a time or fill all tiles within an area, and place them on the map however you fancy. When you are done, make sure to press “stop painting“.
